# ブログプロジェクト - 高品質記事執筆＆管理システム

このプロジェクトは、**包括的なブログレビューシステム**と**自動整理システム**を提供します。

## プロジェクト概要

**目的**: 高品質なブログ記事の執筆・レビュー・公開・管理を効率化

**特徴**:
- 🔍 **12人の専門家による多角的レビュー**
- ⚡ **全自動レビューコマンド（`/full-review`）**
- 📝 **Textlint自動校正（60+パターン）**
- 📁 **記事の自動整理スクリプト**
- 🎯 **3段階のレビューフロー（Think → Write → Review）**

---

## 🚀 クイックスタート - レビューシステム

### 最も推奨される使い方

```bash
# 1. 初稿を書く
vim blog.md

# 2. 全自動レビューを実行（これだけでOK！）
/full-review blog.md コラム

# 3. 8-15分待つ（自動でレビュー実行）

# 4. 統合レポートを確認

# 5. 最優先改善のみ修正を選択

# 6. 記事を整理して公開
python3 organize_blog.py blog.md
git add contents/
git commit -m "feat(blog): 新しい記事を追加"
```

**所要時間**: 約30-60分（レビュー20-40分 + 整理1分 + Git操作1分）

## 全自動レビュー - `/full-review`コマンド

**最重要コマンド**: これ一つですべてのレビューが完了します。

### 実行例

```bash
# コラム記事の場合
/full-review blog.md コラム

# 技術記事の場合
/full-review blog.md 技術
```

### 実行内容（自動→承認→修正の3段階）

**Phase 1: 自動レビュー（8-15分）**
1. Textlint診断（エラー検出のみ、修正は保留）
2. 初学者フィードバック（一般読者視点）
3. 総合品質レビュー（6つの評価軸）
4. 記事タイプ別レビュー（コラム/技術）
5. 統合改善提案レポート生成

**Phase 2: ユーザー承認（ユーザー操作）**
```
📋 レビューが完了しました。

🔴 最優先改善: XX件
🟡 重要改善: XX件
🟢 任意改善: XX件

【ユーザーの選択肢】
1️⃣ 最優先のみ自動修正する（推奨）
2️⃣ 最優先+重要を自動修正する
3️⃣ すべて自動修正する
4️⃣ 詳細レポートを見てから決める
5️⃣ 手動で修正する（自動修正しない）

どれを選択しますか？（1-5）
```

**Phase 3: 承認された修正の適用（1-10分）**
- ユーザーが選択した範囲の修正のみ適用
- 差分確認と最終確認

## 総合品質評価 - 6つの評価軸

**山本理恵**（38歳・テクニカルライター）による評価

### 1. 防御力（Defensibility）: 0.0-5.0点
主張が論理的に防御可能か、反論を想定しているか

### 2. 思考整理力（Structured Thinking）: 0.0-5.0点
論理の流れが明確か、読者が迷わないか

### 3. 実践応用性（Actionability）: 0.0-5.0点
読者が明日から何をすればいいか分かるか

### 4. 構成と読みやすさ（Structure & Readability）: 0.0-5.0点
段落構成、文章のリズム、視覚的な読みやすさ

### 5. コミュニケーション力（Communication Power）: 0.0-5.0点
読者に届くか、共感を呼ぶか、読者との対話があるか

### 6. 人間らしさ（Humanity）: 0.0-5.0点
AI生成ではなく、人間が書いた感じがするか

**総合スコア**: 6つの評価軸の平均（0.0-5.0点）

## 公開推奨度の判定

- **★★★★★ (5/5)**: 今すぐ公開OK、完璧です
- **★★★★☆ (4/5)**: 最優先改善のみ実施して公開OK
- **★★★☆☆ (3/5)**: 重要改善を実施してから公開推奨
- **★★☆☆☆ (2/5)**: 大幅な修正が必要
- **★☆☆☆☆ (1/5)**: 全面的な書き直しを推奨

## Textlint設定 - AIっぽい表現を自動検出

### 検出される問題（60+パターン）

**AIっぽい表現**:
- 冗長な指示詞: 「このような」→「この」
- 過度な強調: 「非常に重要」→「重要」
- テンプレート表現: 「いかがでしたでしょうか」→削除
- 不自然な丁寧語: 「させていただきます」→「します」

**比喩・詩的表現**:
- 「羅針盤」→「指針」
- 「扉を開く」→「開始」
- 「壁にぶつかる」→「困難」

**冗長性**:
- 「することができる」→「できる」
- 「であると言える」→「である」

### 手動実行

```bash
# エラーチェックのみ
npx textlint blog.md

# 自動修正を適用
npx textlint --fix blog.md
```

**注意**: `/full-review`は自動で`--fix`を実行するため、通常は手動実行不要です。

## 記事の自動整理システム

記事のレビューが完了したら、`organize_blog.py`で自動整理します。

### ディレクトリ構造

```
blogs/
├── README.md                 # このファイル
├── CLAUDE.md                 # レビューシステム詳細
├── organize_blog.py         # ブログ整理自動化スクリプト
├── blog.md                  # 作業中のブログ記事
├── .textlintrc.json         # Textlint設定
├── prh.yml                   # 用語統一ルール
├── .claude/commands/        # レビューコマンド
└── contents/                # 整理済みブログ記事
    ├── おい、本を読め/
    │   └── blog.md
    ├── 別の記事/
    │   └── blog.md
    └── ...
```

### 使い方

#### 単一ファイルを整理

```bash
python3 organize_blog.py blog.md
```

これにより、`contents/<タイトル>/blog.md` に記事が移動します。

#### コピーモード（元ファイルを残す）

```bash
python3 organize_blog.py --copy blog.md
```

作業中のファイルを残したい場合は `--copy` オプションを使用します。

#### 複数ファイルを一括整理

```bash
python3 organize_blog.py --all
```

カレントディレクトリ内の全ての `blog*.md` ファイルを自動整理します。

### タイトル抽出ルール

スクリプトは以下のルールでタイトルを抽出します:

1. blog.md の最初の `# ` で始まる行をタイトルとして使用
2. タイトルが見つからない場合はファイル名を使用

### ディレクトリ名生成

タイトルから以下のルールでディレクトリ名を生成します:

- 句読点（、。！？）を削除
- スペースをハイフンに変換
- 連続するハイフンを1つに統合

例:
- `# おい、本を読め` → `おい本を読め/`
- `# Rust の非同期プログラミング入門` → `Rust-の-非同期プログラミング-入門/`

## トラブルシューティング

### Q: `/full-review`が長すぎる場合

**A**: 軽量版を使用
```bash
/full-review blog.md --tier1-only
```
Tier 1（必須）レビューのみ、約5分で完了

---

### Q: Textlintエラーが多すぎる

**A**: 段階的に修正
1. まず`--fix`で自動修正
2. 優先度：高のみ手動修正
3. 優先度：中・低は無視してOK

---

### Q: タイトルが正しく抽出されない

**A**: blog.md の最初の行が `# タイトル` 形式になっているか確認してください。

---

### Q: 移動した記事を元に戻したい

**A**: 以下のコマンドで復元できます:

```bash
cp contents/おい、本を読め/blog.md blog.md
```

## ベストプラクティス

### 記事の長さ

- **推奨**: 2000-3000字
- **最大**: 5000字（それ以上は分割を検討）
- **最小**: 1000字（それ以下は内容不足）

### 文章スタイル

- **基本**: です・ます調
- **例外**: ペルソナの会話文、引用、コードコメント

### AIっぽさの排除

このプロジェクトでは、AIっぽい表現を徹底的に排除します:

**NGワード**:
- 「このような」「そのような」
- 「非常に〜」の多用
- 「いかがでしたでしょうか」
- 「お役に立てれば幸いです」
- 「させていただきます」

**OKな表現**:
- 「この」「その」（簡潔に）
- 適度な強調のみ
- 自然な締め
- 「役立ちます」
- 「します」
