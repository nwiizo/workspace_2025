# Textlint Polish - 文章校正・AIっぽさ除去

このコマンドは、執筆した記事にtextlintを実行し、表現の問題点を検出・修正します。AIっぽい表現、冗長な言い回し、文体の不統一などを自動でチェックします。

## 使用方法

```
/textlint-polish [ファイルパス]
```

例:
```
/textlint-polish blog.md
/textlint-polish draft.md
```

## このコマンドの目的

**機械的・AIっぽい表現を排除し、自然で読みやすい文章にする**

- AIが多用する冗長表現を検出
- 比喩的・詩的すぎる表現を簡潔に
- 文体の統一（です・ます調）
- 技術文書としての品質向上

## ワークフロー

### Step 1: Textlintの実行

ファイルに対してtextlintを実行します。

```bash
cd /Users/nwiizo/ghq/github.com/nwiizo/workspace_2025/blogs
npx textlint [ファイルパス]
```

### Step 2: エラーの分類

検出されたエラーを以下のカテゴリに分類します:

#### カテゴリA: AIっぽい表現（優先度：高）
- 「このような」「そのような」などの冗長な指示詞
- 「非常に」「極めて」などの過度な強調
- 「いかがでしたでしょうか」などのテンプレート的表現

#### カテゴリB: 比喩・詩的表現（優先度：高）
- 「羅針盤」「地図」「風景」などの比喩
- 「扉を開く」「壁にぶつかる」などの慣用表現
- 技術文書には不要な装飾

#### カテゴリC: 冗長性（優先度：中）
- 「することができる」→「できる」
- 「であると言える」→「である」
- 「絶対に必要」→「必要」

#### カテゴリD: 文体の不統一（優先度：中）
- です・ます調とである調の混在
- 句読点の使い方の不統一

#### カテゴリE: 弱い表現（優先度：低）
- 「かもしれない」「だと思います」
- 確信が持てない表現

### Step 3: 自動修正の実行

修正可能なエラーを自動で修正します。

```bash
npx textlint --fix [ファイルパス]
```

### Step 4: 修正結果の確認

自動修正後のdiffを確認し、問題ないかチェックします。

### Step 5: 手動修正が必要な箇所の提案

自動修正できなかった箇所について、具体的な修正案を提示します。

## 使用上の注意

1. **自動修正は慎重に**: `--fix`オプションは便利ですが、必ず差分を確認します
2. **ペルソナの会話文は除外**: 物語調・会話文は意図的な表現なので、必要に応じて無視します
3. **コードブロックは対象外**: Markdown内のコードブロックはtextlintが自動でスキップします
4. **段階的修正**: 優先度の高いものから順に修正し、一度に全て直そうとしない

npx textlint **/*.md
