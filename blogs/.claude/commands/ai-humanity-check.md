# ai-humanity-check

文章のAIっぽさを評価し、より人間らしい表現への改善提案を行います。

## Description

このコマンドは、文章がどれだけAI生成的か、あるいは人間的かを0.0-5.0のスケールで評価します。プロジェクトのマインドセット（特に「勇者ではない主人公」「失敗と成長の哲学」）に基づき、完璧でない人間らしさの価値を重視した評価を行います。

## Usage

```
/ai-humanity-check [text or file to analyze]
```

## 評価スケール

### AIっぽさスコア (0.0-5.0) ※低いほど人間らしい

**0.0-1.0 (完全に人間的)**
- 著者特有の言い回しや癖がある
- 具体的な失敗談や苦労話が生々しい
- 感情の起伏が自然で共感できる
- 予想外の視点や独自の発見がある
- 文章にリズムの変化と個性がある

**1.1-2.0 (ほぼ人間的)**
- 自然な文体だが、時々定型的
- 感情表現は適切だが、深みに欠ける
- オリジナリティは感じるが、驚きは少ない

**2.1-3.0 (AIっぽさが混在)**
- 文体が均一で変化に乏しい
- 具体例が表面的で深掘りされていない
- 感情表現が型通り
- 予測可能な構成と結論
- 「まず」「次に」「最後に」の多用

**3.1-4.0 (明らかにAIっぽい)**
- 機械的で無機質な文体
- 一般論と抽象的な説明が中心
- 過剰な箇条書きと並列構造
- 同じような表現の繰り返し

**4.1-5.0 (完全にAI的)**
- テンプレート的な構成と表現
- 感情や個性が全く感じられない
- 内容が教科書的で面白みがない
- 「以下の通りです」「については」の連発

## 評価フレームワーク

### 1. 文体分析
```yaml
人間的指標:
  - リズムの変化: 短文と長文の自然な混在
  - 話し言葉の混入: 「でも...」「だって」「〜じゃない？」
  - 感情の表出: 「！」「...」の自然な使用
  - 個人的な癖: 特定の言い回しの反復（良い意味で）

AI的指標:
  - 均一な文長: すべて似た長さの文
  - 過度な丁寧さ: 「〜です」「〜ます」の機械的連続
  - 接続詞の多用: 論理的すぎる文章構造
  - 感情の欠如: 事実の羅列のみ
```

### 2. 内容分析
```yaml
人間的要素:
  - 具体的な失敗談: 恥ずかしい、痛い、生々しい
  - 矛盾や迷い: 「でも一方で...」という揺らぎ
  - 個人的な発見: 「気づいたんだけど」
  - 時系列の混在: 過去と現在の行き来

AI的要素:
  - 一般論中心: 「〜と言われています」
  - 完璧な論理: 矛盾のない整然とした説明
  - 抽象的な例: 具体性に欠ける説明
  - 教科書的: 正しいが面白みがない
```

### 3. 感情表現分析
```yaml
人間らしい感情:
  - 生の感情: 「悔しかった」「嬉しくて泣いた」
  - 感情の変化: 一つの文章内での心の動き
  - 未整理な感情: 「なんというか...」
  - 身体的反応: 「胃が痛くなった」「手が震えた」

AI的な感情表現:
  - 定型的: 「喜びを感じました」
  - 感情の羅列: 「嬉しく、楽しく、充実した」
  - 抽象的: 「ポジティブな気持ち」
  - 説明的: 感情を分析的に語る
```

## 出力フォーマット

### 1. スコアカード
```
=== AIっぽさ評価結果 ===
総合スコア: X.X / 5.0
(人間らしさ: ★★★☆☆)

【詳細評価】
文体の自然さ: X.X
感情表現: X.X
具体性: X.X
個性・オリジナリティ: X.X
意外性: X.X

【最も人間的な部分】
"[引用]" - 理由

【最もAI的な部分】
"[引用]" - 理由
```

### 2. 人間らしさの分析
```
💡 人間的な要素:
- 具体的な失敗談がある（例：〜）
- 感情の揺らぎが見える（例：〜）
- 個人的な発見がある（例：〜）

⚠️ AI的な要素:
- 構成が型通り（序論→本論→結論）
- 感情表現が表面的
- 一般論が多い
```

### 3. 改善提案
```
より人間らしくするための提案:

1. 具体的なエピソードを追加
   Before: "失敗から学んだ"
   After: "あの日、本番環境を落として冷や汗が..."

2. 感情の生々しさを加える
   Before: "困難でした"
   After: "正直、もう辞めたいと思った"

3. 矛盾や迷いを残す
   Before: "これが正解です"
   After: "今でも正解かわからないけど..."
```

## 実例

### Example 1: 技術ブログ
```
/ai-humanity-check "本日はReactのuseEffectについて解説します。useEffectは副作用を扱うためのフックで..."
```

評価結果:
- スコア: 3.8/5.0 (明らかにAI的)
- 理由: 教科書的説明、感情なし、「本日は」という定型表現

改善案:
「useEffectで3日間ハマった話をします。最初は簡単だと思ってたんだけど...」

### Example 2: 個人ブログ
```
/ai-humanity-check "転職して3ヶ月。正直、こんなはずじゃなかった。最初の1週間で..."
```

評価結果:
- スコア: 0.8/5.0 (完全に人間的)
- 理由: 生々しい感情、「こんなはずじゃ」という率直さ、時系列の具体性

### Example 3: 自己啓発記事
```
/ai-humanity-check "成功するための5つの習慣を紹介します。まず第一に、早起きです。次に..."
```

評価結果:
- スコア: 4.2/5.0 (完全にAI的)
- 理由: テンプレート構成、「まず」「次に」の多用、一般論のみ

## 哲学的背景

### 勇者ではない主人公の視点
- 完璧でない文章にこそ価値がある
- 失敗や迷いを含む文章が共感を生む
- 小さな発見や気づきが個性になる

### 能力という幻想
- 「良い文章」は文脈依存
- AIっぽさも場面によっては適切
- 読者との関係性で評価は変わる

### 失敗の哲学
- きれいにまとまりすぎた文章は疑われる
- 失敗談や苦労話が信頼を生む
- 「恥をかく勇気」が文章に深みを与える

## 使用上の注意

1. **文脈を考慮**: 技術文書なら多少AI的でもOK
2. **目的に応じて**: 全てを人間的にする必要はない
3. **バランス重視**: 極端に崩しすぎても読みにくい
4. **個性を活かす**: 無理に型を崩す必要はない
