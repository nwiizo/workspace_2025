# Blog Quality Review - ブログ品質レビュー

このコマンドは、ブログ記事を6つの観点から評価します。

## 使用方法

```
/blog-quality-review [ファイルパス]
```

例:
```
/blog-quality-review blog.md
```

## 評価観点

以下の6つの観点（各0.0-5.0スコア）で評価します：

1. **防御力** - 批判や反論への耐性
2. **思考整理力** - 情報の論理的構造化
3. **実践応用性** - 読者が行動に移せる価値
4. **構成と読みやすさ** - 視覚的要素と文体
5. **コミュニケーション力** - 人間味のある伝達
6. **人間らしさ** - 温度感と個性

**山本理恵（やまもと りえ）・38歳**

- 大手IT企業で10年間エンジニアとして勤務後、技術コミュニケーションの道へ

## 評価手順

### Step 1: コンテンツを読む

指定されたファイルを全文読み、以下をメモする：
- ターゲット読者層
- 記事の目的（解説/考察/エッセイなど）
- 著者の意図

### Step 2: 6つの観点で評価

各観点について0.0-5.0でスコアリングし、具体的な根拠を示す。

#### 1. 防御力 (0.0-5.0)

**評価ポイント:**
- 主観と客観の区別が明確か
- 前提条件や制約を明示しているか
- 根拠と出典を適切に提示しているか
- 異なる視点への配慮があるか
- 誠実さと謙虚さが感じられるか

**チェック項目:**
```
□ 「私の経験では」「〜の環境で」など主観的表現を使用
□ 前提条件・制約を明記
□ 批判を先取りし「適さないケース」も説明
□ 知識の限界を認める姿勢
□ 断定的表現を避けている
```

**スコアリング基準:**
- 5.0: 完璧な防御力（全項目クリア、批判への先回り対応）
- 4.0: 高い防御力（主観表現使用、コンテキスト明示、根拠提示）
- 3.0: 標準的（部分的に主観示すが、断言的表現が残る）
- 2.0: 弱い（断言的表現多く、前提条件説明不足）
- 1.0: 非常に弱い（根拠なき断言、独善的）

---

#### 2. 思考整理力 (0.0-5.0)

**評価ポイント:**
- 情報の優先順位と強弱が明確か
- 論理的な流れと構成があるか
- 思考プロセスが可視化されているか
- 抽象と具体のバランスが取れているか
- 結論への道筋が明確か

**チェック項目:**
```
□ 「なぜこのテーマに取り組んだか」が明確
□ 試行錯誤の過程が含まれている
□ 重要な概念は詳しく、補助的情報は簡潔
□ 読者が著者の思考を追体験できる
□ 情報の優先順位が明示されている
```

**スコアリング基準:**
- 5.0: 卓越（問題→過程→結論の明確な流れ、思考の追体験可能）
- 4.0: 優れた（論理的構成、主要論点と補助情報の区別）
- 3.0: 標準的（基本的論理構造あるが、重要度区別曖昧）
- 2.0: 弱い（情報並列羅列、何が重要か不明）
- 1.0: 非常に弱い（断片的、論理的つながりなし）

---

#### 3. 実践応用性 (0.0-5.0)

**評価ポイント:**
- 具体的な手順や方法を提示しているか
- 著者独自の経験や知見があるか
- 実例と応用可能性があるか
- 読者が次に何をすべきか明確か

**チェック項目:**
```
□ 具体例や実例
□ 実際の体験に基づく知見
□ つまずきポイントと解決策
□ 「なぜこの方法を選んだか」の理由
□ 成功例と失敗例の両方を提示
```

**スコアリング基準:**
- 5.0: 極めて高い（全項目クリア、即実践可能）
- 4.0: 高い（具体的手順・実例豊富、経験に基づく有用情報）
- 3.0: 標準的（基本情報あるが、詳細不足）
- 2.0: 低い（抽象的説明多く、行動指針不明確）
- 1.0: 非常に低い（理論・概念説明のみ）

---

#### 4. 構成と読みやすさ (0.0-5.0)

**評価ポイント:**
- 導入部の魅力と明確さ
- 見出しと段落の構成
- 文章のリズムと緩急
- 情報密度の適切さ

**チェック項目:**
```
□ 冒頭で記事の価値を明示
□ 適切な見出し階層（## と ### のみ）
□ 3-5行の短い段落
□ スキミングでも要点が掴める
```

**スコアリング基準:**
- 5.0: 完璧（全項目クリア、視覚的に優れた構成）
- 4.0: 優れた（明確な構造、適切な段落）
- 3.0: 標準的（基本構造あるが、長段落あり）
- 2.0: 弱い（構造不明確、文字の壁）
- 1.0: 非常に弱い（構造なし、極端に長い文章）

---

#### 5. コミュニケーション力 (0.0-5.0)

**評価ポイント:**
- 著者の個性と声の表現
- 読者への共感と配慮
- 専門用語の適切な説明
- 親しみやすさ

**チェック項目:**
```
□ 著者の個性が光る文体
□ 読者の立場に立った説明
□ 適度な口語表現
□ 「一緒に考えよう」という姿勢
□ 専門用語には必ず説明
```

**スコアリング基準:**
- 5.0: 卓越（全項目クリア、強い個性）
- 4.0: 良好（親しみやすい文体、読者への配慮、経験共有）
- 3.0: 標準的（情報伝達できるが、やや事務的）
- 2.0: 弱い（一方的、理解度への配慮不足）
- 1.0: 非常に弱い（読者の存在を意識していない）

---

#### 6. 人間らしさ (0.0-5.0)

**評価ポイント:**
- 文体の自然さと変化
- 感情表現の真実味
- 文章の個性と癖
- 意外性とオリジナリティ

**チェック項目:**
```
□ 著者特有の言い回しや癖がある
□ 具体的な失敗談や苦労話が生々しい
□ 感情の起伏が自然で共感できる
□ 予想外の視点や独自の発見がある
□ 文章にリズムの変化と個性がある
```

**機械的な文章の兆候チェック:**
```
□ 「以下の通りです」「については」の頻出
□ 過度に整った構成
□ 具体的な失敗談の欠如
□ 感情表現の不自然さ
□ 一般論への終始
```

**スコアリング基準:**
- 5.0-4.1: 非常に人間的（特有の癖、生々しい失敗談、自然な感情、意外性）
- 4.0-3.1: 人間的（自然な文体、適切な感情、オリジナリティ）
- 3.0-2.1: やや機械的（均一な文体、表面的具体例、型通りの感情）
- 2.0-1.1: 機械的（無機質、一般論中心、過剰な箇条書き）
- 1.0-0.0: 完全に機械的（テンプレート的、感情なし、教科書的）

## 評価の基本姿勢

1. **建設的** - 批判ではなく、改善への道筋を示す
2. **具体的** - 抽象的な指摘ではなく、行番号と実例を挙げる
3. **実践的** - すぐに実行できる改善案を提供する
4. **共感的** - 著者の努力を認め、励ます
5. **公平** - 良い点も悪い点も偏りなく評価する

---

## 重要な注意事項

- この評価は完璧を求めるものではなく、**継続的な改善を支援するツール**
- 「80%の完成度で公開し、フィードバックを受けて改善する」サイクルを推奨
- 記事の文脈（個人ブログ/企業ブログなど）に応じて柔軟に評価
- AI時代において「人間にしか書けない価値」があるかを特に重視
