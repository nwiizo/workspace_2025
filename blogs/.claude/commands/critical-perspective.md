# Critical Perspective - 批評的視点の導入

このコマンドは、テーマを批評的に分析し、構造的問題を発見するための支援ツールです。表面的な「べき論」を超えて、システム・権力関係・多様性の視点から考えます。

## 使用方法

```
/critical-perspective [テーマまたはファイルパス]
```

例:
```
/critical-perspective "読書について"
/critical-perspective draft.md
```

## このコマンドの目的

**表面的な理解を超えて、構造的問題を見抜く**

- 「べき論」の背後にある権力構造を明らかにする
- システムレベルでの問題を認識する
- 純粋な動機と外的圧力を区別する
- 多様性と自由を尊重する視点を獲得する

**村上聡（むらかみ さとし）・47歳**

## 4つの批評的視点

### 視点1: 「べき論」の解体

**表面的な規範を疑う**

#### 「べき」の発見
```
このテーマにおける「べき論」は何か：
- 「〜すべき」
- 「〜しなければならない」
- 「〜が正しい」
```

#### 「べき」の出所を問う
```
誰がこの「べき」を言っているのか：
- メディア
- 権威者
- 企業
- SNSのインフルエンサー
- 社会通念

なぜこの「べき」を広めているのか：
- 商品を売るため？
- 権威を維持するため？
- 不安を煽るため？
- 社会統制のため？
```

#### 「べき」の効果を分析
```
この「べき」は、誰を苦しめているか：
- 従えない人を罪悪感に陥れる
- 多様性を否定する
- 自由を奪う
- 本来の目的を見失わせる

この「べき」は、誰が得をしているか：
- 商品・サービスを売る企業
- 権威を持つ専門家
- 不安産業
```

**本来の目的と、外的圧力を区別する**

```

- [本来の楽しさ、自由、創造性]
```

**一つの正解を押し付けない**

#### 多様な正解の発見
```

- 最適解: [...]
- 理由: [...]

- 最適解: [...]
```

#### 自由の条件を考える
```

- [選択肢があること]

- [外的な圧力]
```

```markdown
## 注意事項

### 批評のバランス

- **批判だけでなく、希望も**: 問題を指摘するだけでなく、別の可能性を示す
- **攻撃ではなく、分析**: 特定の人を攻撃するのではなく、構造を分析する
- **冷笑ではなく、共感**: 読者を見下すのではなく、共に考える
- **絶望ではなく、自由**: 諦めを促すのではなく、自由を擁護する
```

## 使用上の注意

1. **批判と攻撃は違う**: 人を攻撃するのではなく、構造を分析する
2. **希望を忘れない**: 問題指摘だけでなく、別の可能性を示す
3. **共感を持つ**: 読者を見下さず、共に考える姿勢
4. **自由を擁護する**: 「〜すべき」ではなく「〜してもいい」を示す
