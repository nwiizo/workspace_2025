# Title Workshop - タイトル案生成ワークショップ

このコマンドは、ブログ記事のタイトル案を複数生成し、最適なタイトルを選ぶ支援をします。

## 使用方法

```
/title-workshop [テーマまたはファイルパス]
```

例:
```
/title-workshop "読書についての記事"
/title-workshop blog.md
```

## このコマンドの目的

**クリックされ、読まれ、記憶に残るタイトルを作る**

- 複数の切り口からタイトル案を生成
- SEOと感情訴求の両立
- 記憶に残る言葉選び
- SNSでシェアされやすいタイトル

## タイトル生成の5つのアプローチ

### アプローチ1: 問いかけ型

**読者の疑問を直接タイトルに**

```
「なぜ〜なのか」
「どうすれば〜できるのか」
「〜は本当か」
「〜って、何？」
```

**例**:
- なぜ「読書」は苦痛になったのか
- どうすれば本を「楽しく」読めるのか
- 読書は本当に人を変えるのか

### アプローチ2: 対比・逆説型

**常識を覆す、意外性を出す**

```
「〜ではなく、〜」
「〜のに、〜」
「〜すればするほど、〜」
```

**例**:
- 本を読むな、問いを増やせ
- 読書を勧めるほど、読書から遠ざかる
- 積読が多いほど、本は読まれない

### アプローチ3: 数字・具体型

**具体的な数字や期間で信頼性を**

```
「〜つの方法」
「〜日で〜」
「〜%の人が知らない」
```

**例**:
- 読書を変える3つの問い
- 10年読書を続けて分かった、たった1つのこと
- 90%の人が勘違いしている読書の意味

### アプローチ4: 共感・体験型

**読者の感情に寄り添う**

```
「〜したい人へ」
「〜に疲れたあなたへ」
「〜だった私が、〜になるまで」
```

**例**:
- 読書に疲れたあなたへ
- 「読まなきゃ」に縛られている人へ
- 読書嫌いだった私が、本と向き合うまで

### アプローチ5: 宣言・断定型

**強いメッセージで引き込む**

```
「〜すべきだ」
「〜してはいけない」
「〜が全てを変える」
```

**例**:
- おい、本を読め
- 本棚を増やすな、問いを増やせ
- 読書は、答えではなく問いを与える

## タイトル生成ワークフロー

### Step 1: 記事の核心を抽出

**記事の内容を分析**

```
コアメッセージ: [一言で]
読者の問題: [何に困っているか]
提供価値: [何が得られるか]
感情のトーン: [希望/共感/挑発/静謐]
```

### Step 2: 5つのアプローチで各5案生成

**合計25案を生成**

```
問いかけ型:
1. [...]
2. [...]
3. [...]
4. [...]
5. [...]

対比・逆説型:
1. [...]
...

（全5カテゴリ × 5案 = 25案）
```

### Step 3: TOP 10を選定

**以下の基準で評価**

```
評価軸:
- クリック欲求（5点満点）: タイトルを見て、読みたくなるか
- 内容整合性（5点満点）: 記事の内容と合致しているか
- 記憶残存性（5点満点）: 読後、このタイトルを思い出すか
- SNS適性（5点満点）: シェアされやすいか
- SEO適性（5点満点）: 検索されやすいキーワードを含むか

総合スコア: XX/25点
```

### Step 4: 最終推奨案の提示

**TOP 3を推奨理由付きで**

```
🥇 第1候補: [タイトル]
理由: [なぜこれが最適か]
強み: [...]
弱み: [...]
総合スコア: XX/25点

🥈 第2候補: [タイトル]
...

🥉 第3候補: [タイトル]
...
```

## タイトル作成のDo & Don't

### ✅ Do（すべきこと）

- 30文字以内に収める
- 核心的なメッセージを入れる
- 読者の好奇心を刺激する
- 記事の内容と整合性を保つ
- 声に出して読んでみる

### ❌ Don't（避けるべきこと）

- 釣りタイトル（内容と乖離）
- 長すぎるタイトル（40文字超）
- 抽象的すぎるタイトル
- ありきたりな表現
- 専門用語の多用
