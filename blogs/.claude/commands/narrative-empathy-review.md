# Narrative Empathy Review - 物語構造・共感設計レビュー

このコマンドは、指定されたコンテンツを「物語構造」「エモーショナル・ジャーニー」「読者共感」の観点から分析し、感情的つながりを深める提案を提供します。

## 使用方法

```
/narrative-empathy-review [ファイルパス]
```

例:
```
/narrative-empathy-review blog.md
```

## 評価観点

以下の観点で評価します：

1. **物語構造（Narrative Arc）** - 起承転結が機能しているか
2. **エモーショナル・ジャーニー** - 読者の感情の旅路が設計されているか
3. **共感ポイント** - 読者が「自分のことだ」と感じる箇所
4. **読者変容** - 読む前と後で読者がどう変わるか
5. **感情的解像度** - 読後に残る感情の質と深さ
6. **読後感（Afterglow）** - 本を閉じた瞬間に残る余韻

## 評価手順

### Step 1: コンテンツを読む

指定されたファイルを全文読み、以下をメモする：
- 全体の物語構造（起承転結）
- 読者の感情の流れ（エモーショナル・ジャーニー）
- 共感を呼ぶ具体的な箇所

### Step 2: 物語構造の分析

#### 起承転結の評価

**起：問題提起**
- 読者の心を掴む導入か
- 問題提起は切実か
- 感情的フックがあるか
- スコア: X.X/5.0

**承：深化・展開**
- 読者の理解と感情が深まるか
- 新しい視点が提供されるか
- 感情的な波があるか
- スコア: X.X/5.0

**転：転換・発見**
- 読者に「気づき」が訪れるか
- 感情的な転換点があるか
- 希望や解決への道筋が見えるか
- スコア: X.X/5.0

**結：統合・解決**
- 読者に満足感や達成感があるか
- 感情的な解像度（closure）が得られるか
- 次への行動意欲が湧くか
- スコア: X.X/5.0

### Step 3: エモーショナル・ジャーニーの可視化

読者の感情状態を、記事の各部分で追跡します。

**感情の種類:**
- 絶望・孤独・混乱・不安（ネガティブ）
- 安堵・希望・発見・自信・解放感・決意（ポジティブ）
- 共感・納得・驚き・感動（エンゲージメント）

**感情の転換点:**
どの箇所で読者の感情が大きく変わるか、行番号と共に特定します。

### Step 4: 共感ポイントの特定

読者が「これ、自分のことだ」と感じる箇所を、TOP 5-7として特定します。

**共感の3要素:**
1. **体験の共有** - 「私もそうだった」
2. **感情の共有** - 「その気持ち、分かる」
3. **価値観の共有** - 「そう、そうなんだよ」

### Step 5: 読者変容の描写

**読む前の読者:**
- どんな状態か（感情・認識・行動）

**読んだ後の読者:**
- どう変わるか（感情・認識・行動）

**変容のメカニズム:**
どのような段階を経て変容が起こるか

### Step 6: 感情的解像度と読後感の評価

**感情的解像度（Emotional Resolution）:**
- 読後、読者の心にどんな感情が残るか
- 未解決の感情はないか
- 心地よい余韻があるか

**読後感（Afterglow）:**
- 記事を読み終えた瞬間、読者は何を感じるか
- 次の日も心に残るか
- 人に話したくなるか

## 評価の基本姿勢

1. **読者の心の動きを追う** - 情報ではなく、感情の流れを見る
2. **共感ポイントを発掘する** - 偶然ではなく、設計された共感を特定
3. **具体的に示す** - 抽象的な「面白い」ではなく、「なぜ心が動くか」を分析
4. **物語として読む** - エッセイも一つの「変容の物語」
5. **読後感を重視する** - 情報量ではなく、心に残る余韻が真の価値

---

## 重要な注意事項

- 物語構造は「起承転結がある」だけでは不十分。読者の感情が動くことが重要
- 共感は「情報の共有」ではなく、「体験・感情・価値観の共有」
- 読者変容は知識の増加だけでなく、感情・認識・行動の変化を含む
- 読後感は記事の真の価値。情報は忘れても、感情は残る
