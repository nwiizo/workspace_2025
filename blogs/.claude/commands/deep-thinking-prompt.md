# Deep Thinking Prompt - 深く考えるための問いかけ

このコマンドは、ブログを書く前に「深く考える」ための問いかけを提供します。表面的な理解や一般論で終わらず、本質に迫るための思考支援ツールです。

## 使用方法

```
/deep-thinking-prompt [テーマまたはファイルパス]
```

例:
```
/deep-thinking-prompt "読書について"
/deep-thinking-prompt draft.md
```

## このコマンドの目的

**「何を書くか」ではなく「なぜ書くのか」を明確にする**

- 自分の経験と思考を深く掘り下げる
- 表面的な理解から本質的な洞察へ
- 一般論ではなく、自分だけの視点を見つける
- 読者に届く「問い」を発見する

## 深く考えるための7つの問いかけ

### 1. 原体験への問いかけ

**なぜこのテーマに興味を持ったのか？**

- 最初にこのテーマに触れたのはいつですか？
- そのとき、何を感じましたか？
- その体験は、今のあなたにどう影響していますか？
- 他の人と違う、あなた固有の体験は何ですか？

### 2. 前提への問いかけ

**当たり前だと思っていることは何か？**

- このテーマについて、疑っていないことは何ですか？
- 「常識」として受け入れていることは？
- もしその前提が間違っていたら、何が変わりますか？
- 10年前の常識と今の常識、何が変わりましたか？

### 3. 対立への問いかけ

**矛盾や葛藤はどこにあるか？**

- このテーマについて、自分の中で矛盾していることは？
- 理想と現実のギャップは？
- 「べき論」と「実際」の乖離は？
- 賛成意見と反対意見、どちらも理解できる部分は？

### 4. 構造への問いかけ

**システムとしてどう機能しているか？**

- このテーマを、どんな要素に分解できますか？
- 要素同士はどう関係していますか？
- どんなフィードバックループがありますか？
- レバレッジポイント（変化を起こす急所）はどこですか？

### 5. 時間軸への問いかけ

**過去・現在・未来でどう変化しているか？**

- 過去はどうだったか？
- 現在はどう変化しているか？
- 未来はどうなりそうか？
- あなた自身の変化は？（5年前、今、5年後）

### 6. 他者への問いかけ

**異なる立場の人はどう見ているか？**

- 賛成する人は、なぜ賛成するのか？
- 反対する人は、なぜ反対するのか？
- 無関心な人は、なぜ無関心なのか？
- あなたとは違う価値観の人は、どう考えるか？

### 7. 本質への問いかけ

**結局、何が問題なのか？何を伝えたいのか？**

- このテーマの本質は、一言で言うと？
- 最も重要な問いは何ですか？
- 読者に何を感じてほしいですか？
- この記事を読んだ後、読者にどう変わってほしいですか？

## ワークフロー

### Step 1: テーマの明確化

**ユーザーが提示したテーマを整理する**

```
テーマ: [ユーザーの入力]
想定読者: [推測]
なぜこのテーマか: [ユーザーに問いかける]
```

### Step 2: 7つの問いかけを実行

各問いについて、2-3個の具体的な質問を投げかけます。

**重要**: 答えを求めるのではなく、考えるきっかけを提供します。

### Step 3: 思考の深化マップ作成

```
## 思考の深化マップ

### 表層の理解
- [一般的な理解、常識]

### 中層の洞察
- [自分の経験から得た気づき]

### 深層の本質
- [問いと葛藤の核心]

### あなただけの視点
- [他の人にはない、あなた固有の発見]
```

### Step 4: 執筆への橋渡し

**次のアクションを提案する**

```
## 推奨される執筆アプローチ

1. **導入部**: [どんな問いから始めるか]
2. **本論の軸**: [何を中心に展開するか]
3. **読者への価値**: [読者が得られるもの]
4. **あなたの立ち位置**: [賛成？反対？中立？葛藤？]

## 追加で考えるべき問い

- [さらに深めるための問い]
- [見落としている視点]
```

## 使用上の注意

1. **答えを急がない**: このコマンドは答えを出すためのものではありません
2. **一度で終わらない**: 何度も繰り返し、思考を深めます
3. **対話する**: 一人で考えるだけでなく、誰かと話します
4. **書き留める**: 浮かんだ考えをすぐにメモします
