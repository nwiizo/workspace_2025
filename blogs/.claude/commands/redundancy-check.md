# Redundancy Check - 冗長性チェック

このコマンドは、指定されたコンテンツを冗長性の観点から4つの軸で分析し、削減・簡潔化の具体的提案を提供します。

## 使用方法

```
/redundancy-check [ファイルパス]
```

例:
```
/redundancy-check blog.md
```

## 評価観点

以下の4つの観点（各0.0-5.0スコア）で評価します：

1. **情報密度** - 1文あたりの情報量
2. **簡潔性** - 冗長表現・無駄な修飾の少なさ
3. **論理効率** - 論理的重複・循環論法の少なさ
4. **構造最適性** - 章・節の構成の必要十分性

---

## 評価手順

### Step 1: コンテンツを読む

指定されたファイルを全文読み、以下をメモする：
- 全体の文字数
- 主要な論点の数
- 繰り返しのパターン

### Step 2: 3回読み法で分析

#### 第1回：情報密度チェック
各段落を読み、「この段落の核心は1文で言うと何か？」を問います。
1文で言えることを5行で書いていたら、それは冗長です。

#### 第2回：削除シミュレーション
各文を削除してみて、「意味が通じなくなるか？」をテストします。
通じるなら、その文は不要です。

#### 第3回：構造俯瞰
章・節レベルで「この構成は最短経路か？」を問います。
統合・削除可能な要素を特定します。

### Step 3: 4つの観点で評価

#### 1. 情報密度 (0.0-5.0)

**評価ポイント**: 1文あたり何ビットの情報を含んでいるか

**スコアリング基準**:
- 5.0: 全ての文が新しい情報を提供、削れる文がゼロ
- 4.0: ほぼ全ての文が必要、やや装飾的表現あり
- 3.0: 半分は有用だが、半分は削除可能
- 2.0: 同じ内容の繰り返しが目立つ
- 1.0: ほとんどが冗長、本質は1/3で伝えられる

---

#### 2. 簡潔性 (0.0-5.0)

**評価ポイント**: 冗長表現・無駄な修飾の少なさ

**悪い例**:
- "〜ということができると考えられます" → "〜です"
- "〜について説明したいと思います" → "〜を説明します"
- "非常に重要であると言えます" → "重要です"

**スコアリング基準**:
- 5.0: 一切の贅肉なし、全ての語が機能している
- 4.0: わずかな冗長表現（1-2箇所）
- 3.0: 頻繁に冗長表現が出現（10箇所以上）
- 2.0: 文章の50%が冗長表現
- 1.0: 官僚的・AI的な冗長表現だらけ

---

#### 3. 論理効率 (0.0-5.0)

**評価ポイント**: 論理的重複・循環論法の少なさ

**チェック項目**:
```
□ 同じ論点を別の言葉で繰り返していないか
□ AとBの例が実は同じ構造ではないか
□ 「つまり」の後が「つまり」になっていないか（本当に要約か）
□ 結論が既に前提に含まれていないか（循環論法）
□ セクション間で論理が重複していないか
```

**スコアリング基準**:
- 5.0: 論理展開が最短経路、重複ゼロ
- 4.0: わずかな論理重複（1箇所）
- 3.0: 複数の論理重複（3-5箇所）
- 2.0: 頻繁な論理重複、循環論法
- 1.0: 論理が堂々巡り、進展がない

---

#### 4. 構造最適性 (0.0-5.0)

**評価ポイント**: 章・節の構成は必要十分か

**チェック項目**:
```
□ この章は独立した価値があるか（他章に統合できないか）
□ このセクションは独立した価値があるか（削除/統合できないか）
□ 見出しの階層が深すぎないか（3階層以内が理想）
□ 例示は必要最小限か（3つ→2つ、2つ→1つにできないか）
□ 図表は本当に必要か（テキストで十分では）
```

**スコアリング基準**:
- 5.0: 完璧な構成、削れる要素がゼロ
- 4.0: ほぼ最適、わずかな統合余地
- 3.0: 統合可能なセクションが複数
- 2.0: 冗長な構造、30%削減可能
- 1.0: 構造的に破綻、50%以上削減可能

## 評価の基本姿勢

1. **容赦なく削る** - 迷ったら削除を提案
2. **読者の時間を守る** - 冗長性は読者への不誠実
3. **構造的視点** - 木を見て森も見る
4. **具体的提案** - 抽象的指摘ではなく、具体的な削除/統合案を提示

---

## 重要な注意事項

- **削りすぎない**: 読みやすさのための適度な「余白」は必要
- **文体を壊さない**: 著者の声（voice）は尊重する
- **文脈を考慮**: 本の序盤は丁寧、後半は簡潔、など戦略的冗長性もある
- **数値目標ではない**: 「XX%削減」が目的ではなく、「最適な情報密度」が目的
